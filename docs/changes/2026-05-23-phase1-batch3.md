**Type:** `feat`
**Commit:** _(pending)_

## Context

Third (and last) Phase 1 batch from the UX audit. Item #5 (smart defaults from last game) was skipped per user direction. Items #10 (date/time pickers), #9 (zone-tap UX cleanup), and #4 (`is_home_game` recovery) all landed here.

Findings addressed: `UX-NG-08`, `UX-NG-09`, `UX-LG-02`, `UX-LG-03`, `UX-LG-12`.

## Plan (Decisions)

- **Mobile date/time pickers** — install `@react-native-community/datetimepicker` (not on the iOS 26.2 beta blocklist, no TurboModule deps). Two `<Pressable>` buttons that open the native picker. Single `Date` state internally; `handleCreate` uses `.toISOString()` directly.
- **Zone-tap UX** — keep two-tap behavior intact, add two new affordances in `live.tsx`: a phase-aware subtitle ("Tap target zone" → "Tap actual location" → "✓ Target + actual set"), and an outlined "Pitch hit target ({zone})" button that one-taps `pitchLocation` from the target zone's center. No changes to the StrikeZone component (still shared with bullpen/breakdown).
- **Home/away recovery** — server-side `gameService.toggleHomeAway` is a clean one-field flip; scoring convention keeps opponent in `away_score` and user in `home_score` regardless of `is_home_game`. Enable the swap post-pitches with a confirm dialog explaining the implication.

Full plan: [`docs/plans/2026-05-23-phase1-batch3.md`](../plans/2026-05-23-phase1-batch3.md).
